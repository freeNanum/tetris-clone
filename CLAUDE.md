# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code(및 다른 AI 코딩 어시스턴트)를 위한 가이드입니다.

## 프로젝트 개요

웹 브라우저에서 즉시 플레이 가능한 **캐주얼 테트리스 게임**입니다.
설치나 빌드 과정 없이 `index.html`을 열면 바로 실행되는 것을 목표로 합니다.

- 대상: 가볍게 즐기는 캐주얼 게이머 (복잡한 랭킹/멀티플레이 없음)
- 플랫폼: 데스크톱 브라우저 우선, 모바일 터치도 지원
- 톤: 심플하고 밝은 캐주얼 비주얼, 부담 없는 난이도 곡선

## 기술 스택

- **HTML5 + CSS3 + Vanilla JavaScript (ES6+)** — 프레임워크/빌드 도구 없음
- 렌더링: **Canvas API** (2D context)
- 오디오: Web Audio API 또는 `<audio>` 태그 (효과음/BGM)
- 저장: `localStorage` (최고 점수, 설정 저장)
- 배포: 정적 파일 그대로 로컬 실행 또는 GitHub Pages 등 정적 호스팅

빌드 도구, 패키지 매니저, 프레임워크를 추가하지 않습니다. 의존성이 필요할 것 같으면 먼저 사용자와 상의합니다.

## 프로젝트 구조 (예정)

```
/Tetris
├── index.html          # 진입점, 캔버스와 UI 마크업
├── css/
│   └── style.css       # 레이아웃, 캐주얼 테마 스타일
├── js/
│   ├── main.js         # 진입점, 게임 루프 시작
│   ├── board.js         # 보드(그리드) 상태 관리
│   ├── piece.js         # 테트로미노 정의, 회전/이동 로직
│   ├── game.js          # 게임 상태 머신 (대기/플레이/일시정지/게임오버)
│   ├── input.js         # 키보드/터치 입력 처리
│   ├── renderer.js      # Canvas 렌더링 담당
│   ├── audio.js         # 사운드 재생
│   └── storage.js       # localStorage 래퍼 (최고 점수 등)
├── assets/
│   ├── sfx/             # 효과음
│   └── fonts/           # 커스텀 폰트 (선택)
└── CLAUDE.md
```

파일명과 구조는 개발 중 조정될 수 있습니다. 실제 구조는 코드가 만들어진 뒤 이 문서를 갱신하세요.

## 코딩 컨벤션

- ES6+ 모듈(`import`/`export`) 사용, `<script type="module">`로 로드
- 클래스보다는 순수 함수 + 명시적 상태 객체를 우선 (테스트와 디버깅이 쉬움)
- 게임 상태는 하나의 단일 객체(state)로 관리하고, 렌더링은 상태를 읽기만 함 (상태와 렌더링 분리)
- 매직 넘버(보드 크기, 낙하 속도 등)는 `constants.js` 또는 파일 상단에 상수로 정의
- 주석은 "왜"가 비직관적일 때만 작성 (예: SRS 회전 보정, 락 딜레이 이유 등)

## 개발 명령어

빌드 도구가 없으므로 별도 빌드 명령이 없습니다.

- 로컬 실행: `index.html`을 브라우저로 직접 열기, 또는
  ```
  npx serve .
  ```
  같은 간단한 정적 서버로 실행 (CORS/module 이슈 방지용)
- 테스트: 브라우저에서 직접 플레이하며 확인 (자동화 테스트는 3단계 이후 논의)

## 핵심 게임 규칙 (참고)

- 보드 크기: 10 x 20 (표준)
- 7종 테트로미노: I, O, T, S, Z, J, L
- 랜덤 배출: 7-bag 랜덤(한 세트 7개가 모두 나온 뒤 다음 세트 셔플)을 기본으로 채택 — 캐주얼 게임에서도 "운이 지나치게 나쁜" 경험을 줄여줌
- 라인 클리어 점수, 레벨업에 따른 낙하 속도 증가는 [Tetris Guideline](https://tetris.wiki/Tetris_Guideline)을 단순화해서 참고

## 작업 시 주의사항

- 캐주얼 게임이므로 지나치게 복잡한 룰(예: T-spin 세부 판정, 홀드 큐 다단계)은 초반 단계에서 생략하고 후반에 옵션으로 추가
- 성능: `requestAnimationFrame` 기반 루프를 사용하고, 매 프레임 불필요한 객체 생성을 피함
- 접근성: 키 리매핑, 터치 버튼 크기 등 캐주얼 사용자를 고려
