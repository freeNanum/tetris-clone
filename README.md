# 캐주얼 테트리스 (Casual Tetris)

브라우저에서 설치나 빌드 과정 없이 바로 즐기는 캐주얼 테트리스 게임입니다.
`index.html`을 열면 바로 플레이할 수 있습니다.

## 빠른 시작

빌드 도구가 필요 없습니다. 아래 방법 중 하나를 사용하세요.

### 방법 1 — 파일을 바로 열기

`index.html` 파일을 더블클릭하거나 브라우저의 "파일 열기"로 실행합니다.

> 일부 브라우저는 `file://` 경로에서 ES Modules(`<script type="module">`)를 제한할 수 있습니다.
> 이 경우 방법 2를 사용하세요.

### 방법 2 — 간단한 정적 서버로 실행 (권장)

```bash
# Node.js가 있는 경우
npx serve .

# 또는 Python이 있는 경우
python -m http.server 8080
```

터미널에 안내된 주소(예: `http://localhost:8080`)를 브라우저에서 열면 됩니다.

## 조작법

### 키보드

| 동작 | 키 |
|---|---|
| 좌/우 이동 | `←` `→` (또는 `A` `D`) |
| 소프트 드롭(빨리 내리기) | `↓` (또는 `S`) |
| 하드 드롭(즉시 낙하) | `Space` |
| 시계방향 회전 | `↑` (또는 `X`) |
| 반시계방향 회전 | `Z` |
| 홀드 | `C` (또는 `Shift`) |
| 일시정지 | `P` (또는 `Esc`) |

### 모바일 / 터치

화면 폭이 좁거나 터치 기기로 접속하면 화면 하단에 터치 버튼이 나타납니다.

- `◀` `▶` — 좌/우 이동 (누르고 있으면 반복 이동)
- `▼` — 소프트 드롭 (누르고 있는 동안 빠르게 낙하)
- `⟳` — 회전
- `HOLD` — 홀드
- `DROP` — 하드 드롭

## 게임 규칙

- 보드 크기: 10 x 20
- 조각 배출: **7-bag 랜덤** — 7종 조각(I, O, T, S, Z, J, L)이 한 번씩 모두 나온 뒤 다음 세트가 셔플되어 나옵니다. 특정 조각이 지나치게 안 나오는 "운 나쁜" 경험을 줄여줍니다.
- 점수: 동시에 지운 줄 수에 따라 보너스가 커집니다 (1줄 100 / 2줄 300 / 3줄 500 / 4줄 800, 레벨에 비례해 가중치 적용). 하드 드롭·소프트 드롭 거리에도 추가 점수가 붙습니다.
- 레벨: 10줄을 지울 때마다 레벨이 올라가고 자동 낙하 속도가 빨라집니다.
- 최고 점수는 브라우저 `localStorage`에 저장되어 새로고침 후에도 유지됩니다.
- 락 딜레이: 조각이 바닥(또는 다른 블록)에 닿은 뒤 약 0.5초의 여유 시간이 있어 마지막 순간에 좌우로 밀어넣을 수 있습니다.

## 주요 기능

- 다음 조각 3개 미리보기 (Next Queue)
- 홀드(Hold) — 한 조각당 한 번만 교체 가능
- 고스트 피스 — 현재 조각이 떨어질 위치를 미리 표시
- 시작 / 일시정지 / 게임 오버 화면 전환
- Web Audio API로 합성한 효과음 (별도 음원 파일 불필요)
- 모바일 터치 컨트롤 및 반응형 레이아웃

## 기술 스택

- **HTML5 / CSS3 / Vanilla JavaScript (ES6+ 모듈)** — 프레임워크·빌드 도구 없음
- **Canvas 2D API** — 보드/조각 렌더링
- **Web Audio API** — 효과음 합성
- **localStorage** — 최고 점수 저장

의존성이나 빌드 파이프라인이 없어 프로젝트 폴더를 그대로 복사하거나 정적 호스팅(GitHub Pages 등)에 올리기만 하면 동작합니다.

## 아키텍처

- **상태와 렌더링 분리**: `game.js`가 순수한 게임 상태(`state`)를 관리하고, `renderer.js`는 그 상태를 읽어 그리기만 합니다.
- **콜백 기반 이벤트**: `createGame(callbacks)`에 `onMove`, `onLineClear`, `onGameOver` 등 콜백을 넘겨 사운드 재생/HUD 갱신을 트리거합니다 (`main.js` 참고).
- **게임 루프**: `requestAnimationFrame` 기반으로 매 프레임 `game.update(dt)` → `renderer.render(state, ghost)` 순으로 호출합니다.
- **입력 처리**: `input.js`가 키보드/터치 이벤트를 게임 메서드(`moveLeft`, `rotate`, `hold` 등) 호출로 변환합니다.

## 알려진 단순화 사항

캐주얼 게임 목적에 맞춰 일부 규칙을 의도적으로 단순화했습니다.

- **벽킥(Wall Kick)**: 공식 SRS의 5-test 킥 테이블 대신, 좌우/위쪽 오프셋을 순차 시도하는 축약판을 사용합니다. T-spin 등 고급 회전 판정은 지원하지 않습니다.
- **키 반복(DAS)**: 좌우 이동은 OS/브라우저의 기본 키 반복 속도를 그대로 사용합니다. 정식 테트리스처럼 세밀한 DAS(자동 반복 지연) 튜닝은 하지 않았습니다.
- **효과음만 지원**: 배경음악(BGM)은 포함하지 않았고, 효과음은 오디오 파일 없이 코드로 합성합니다.

## 브라우저 지원

최신 Chrome, Edge, Firefox, Safari에서 테스트되었습니다. ES Modules와 Web Audio API를 지원하는 브라우저가 필요합니다.

## 트러블슈팅

- **효과음이 안 들려요**: 브라우저 정책상 사용자가 화면을 클릭하거나 키를 입력하기 전까지 오디오가 잠겨 있습니다. "시작하기" 버튼을 누르거나 아무 키를 입력하면 정상적으로 소리가 납니다.
- **`index.html`을 직접 열었을 때 화면이 비어 있어요**: 브라우저 콘솔에 CORS/module 관련 오류가 있는지 확인하고, [빠른 시작](#빠른-시작)의 방법 2(정적 서버)로 실행해보세요.
- **최고 점수가 초기화돼요**: 시크릿/프라이빗 브라우징 모드에서는 `localStorage`가 세션 종료 시 삭제됩니다.
