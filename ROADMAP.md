# 개발 로드맵 — 웹 캐주얼 테트리스

각 단계는 "실행해서 눈으로 확인 가능한" 결과물을 목표로 합니다. 순서대로 진행하되, 막히면 이전 단계로 돌아가도 됩니다.

## 0단계 — 뼈대 세팅
- `index.html`, `css/style.css`, `js/main.js` 생성
- 화면에 고정 크기 Canvas(예: 300x600) 하나 띄우기
- `requestAnimationFrame` 게임 루프 뼈대만 작성 (매 프레임 배경만 그리기)
- **완료 기준**: 브라우저에서 빈 캔버스가 60fps로 렌더링됨

## 1단계 — 보드와 정적 블록
- 10x20 그리드 데이터 구조(`board.js`) 설계
- Canvas에 그리드 라인, 셀 색상 그리기 (`renderer.js`)
- 보드에 임의의 셀을 채워서 렌더링 테스트
- **완료 기준**: 보드 격자가 화면에 정확히 그려지고, 셀 하나를 채우면 색이 보임

## 2단계 — 테트로미노 정의와 낙하
- 7종 테트로미노 모양/색상 정의 (`piece.js`)
- 랜덤 배출 로직 (7-bag)
- 조각이 일정 간격으로 한 칸씩 아래로 이동 (자동 낙하)
- 바닥/다른 블록에 닿으면 보드에 고정(lock)
- **완료 기준**: 조각이 위에서 아래로 떨어지고 바닥에 쌓임

## 3단계 — 입력 처리
- 키보드 입력(`input.js`): 좌/우 이동, 소프트 드롭, 하드 드롭, 회전(CW/CCW)
- 벽/충돌 체크 (이동·회전 시 보드 경계 및 다른 블록과 겹침 방지)
- 간단한 SRS 스타일 벽킥(wall kick) 또는 단순 회전 보정
- **완료 기준**: 키보드로 조각을 자유롭게 이동/회전/드롭 가능

## 4단계 — 라인 클리어와 점수
- 가로 한 줄이 가득 차면 삭제 + 위 블록 내려앉기
- 점수 계산 (1/2/3/4줄 동시 클리어 보너스 포함)
- 레벨 시스템: 일정 줄 수마다 레벨업 → 낙하 속도 증가
- 최고 점수 `localStorage` 저장/불러오기
- **완료 기준**: 줄을 채우면 삭제되고 점수/레벨이 화면에 반영됨

## 5단계 — 게임 상태 관리
- 상태 머신: 시작 화면 → 플레이 → 일시정지 → 게임 오버 → 재시작
- 게임 오버 조건(스폰 위치가 막힘) 처리
- UI 오버레이(시작/일시정지/게임오버 화면) HTML/CSS로 구성
- **완료 기준**: 처음부터 끝까지 한 판을 완주할 수 있음 (시작 → 플레이 → 게임오버 → 재시작)

## 6단계 — 캐주얼 편의 기능
- 다음 조각 미리보기 (Next queue)
- 홀드(Hold) 기능
- 고스트 피스(낙하 위치 미리보기)
- 소프트/하드 드롭 시각 효과
- **완료 기준**: 캐주얼 테트리스에서 기대하는 기본 편의 기능이 모두 동작

## 7단계 — 비주얼/오디오 폴리싱
- 캐주얼한 색감/폰트/애니메이션(블록 착지, 라인 클리어 이펙트)
- 효과음(이동, 회전, 클리어, 게임오버) + 배경음악
- 화면 흔들림 등 작은 피드백 효과 (선택)
- **완료 기준**: 플레이 경험이 "가볍고 즐겁게" 느껴짐

## 8단계 — 모바일/반응형 대응
- 터치 버튼(좌/우/회전/드롭) UI 추가
- 화면 크기에 따른 캔버스 반응형 스케일링
- **완료 기준**: 모바일 브라우저에서도 불편함 없이 플레이 가능

## 9단계 — 마무리
- 다양한 브라우저(Chrome/Edge/Safari/Firefox)에서 테스트
- 버그 수정, 성능 점검 (프레임 드랍, 메모리 누수)
- README 작성, 정적 호스팅(GitHub Pages 등) 배포
- **완료 기준**: 누구나 링크만으로 바로 플레이 가능

---

## 진행 방식 제안
- 단계별로 완료 기준을 눈으로 확인한 뒤 다음 단계로 넘어가는 것을 권장합니다.
- 0~5단계가 "플레이 가능한 최소 버전(MVP)"이고, 6단계 이후는 완성도를 높이는 단계입니다.
- 각 단계 시작 전 간단히 알려주시면 해당 단계만 집중해서 구현하겠습니다.
